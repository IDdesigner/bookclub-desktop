This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A teacher-facing desktop web app for creating AI-tutored reading assignments. Teachers manage books, classes, and assignments; students interact with an AI tutor that grades comprehension against rubrics. The entire frontend lives in `teacher-app/`.

## Commands

All commands run from `teacher-app/`:

```bash
npm run dev       # Start Vite dev server
npm run build     # Type-check + production build (tsc -b && vite build)
npm run lint      # ESLint
npm run preview   # Serve the production build locally
npm run test      # Vitest in watch mode
npm run test:run  # Vitest single run (CI)

# Run a single test file
npx vitest run src/__tests__/stores/classStore.test.ts
```

## Testing

**Stack**: Vitest + Testing Library (jsdom environment). `globals: true` is set so `describe`/`it`/`expect` are available without imports.

**Folder structure** (all tests live under `src/__tests__/`):

```
src/__tests__/
├── setup.ts        # Global setup — imports @testing-library/jest-dom matchers
├── components/     # Component render tests (@testing-library/react)
├── hooks/          # Custom hook tests (renderHook)
├── stores/         # Zustand store action/state tests
└── utils/          # Pure function tests (e.g., generateClassCode)
```

When testing Zustand stores, reset store state between tests using the store's `setState` or by calling reset actions to avoid cross-test bleed. When testing components that depend on Supabase, mock `src/lib/supabase.ts` with `vi.mock`.

## Architecture

### State Management Split

State is split between two concerns:

- **Zustand stores** (`src/stores/`) for domain state that must persist across route changes — `classStore.ts` owns class/student data, `assignmentStore.ts` owns assignments/rubrics. These stores call Supabase directly.
- **TanStack Query** (`QueryClientProvider` is wired in `App.tsx`) is available but not yet used for data fetching.
- **React Hook Form** handles all form state locally within components.

### Data Flow

Components call Zustand store actions → stores call Supabase client (`src/lib/supabase.ts`) directly → UI re-renders via Zustand subscriptions. There is no service/repository abstraction layer between stores and Supabase.

### Routing & Auth

`App.tsx` defines all routes. Protected routes are wrapped in `<ProtectedRoute>` which reads from the `useAuth` hook (`src/hooks/useAuth.ts`). Auth state comes from Supabase Auth listeners. The layout shell (`src/components/layout/Layout.tsx`) wraps all protected pages with `<Sidebar>` + `<Outlet>`.

Route map:
- `/login` — public
- `/` — Dashboard
- `/books` — Books (placeholder)
- `/classes` — Classes list
- `/classes/:classId` — ClassDetails (Roster + Assignments tabs)
- `/analytics` — Analytics (placeholder)

### Type System

All Supabase table shapes are defined in `src/types/database.types.ts`. When adding new tables or columns, update this file first.

### Backend: Supabase

- **Auth**: email/password via Supabase Auth
- **Database**: PostgreSQL with Row Level Security (teachers own their classes; policies are in `Knowledge Bank/*.sql`)
- **Storage**: planned for book content
- **Edge Functions**: planned for AI tutor sessions

Environment variables required (see `teacher-app/.env.example`):
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

## Key Domain Concepts

**Assignment lifecycle**: assignments are created in `draft` status, then published to make them visible to students. The `unpublish` action returns them to draft. This is managed in `assignmentStore.ts`.

**Class invite codes**: 8-character alphanumeric codes in `XXXX-XXXX` format, generated by `src/utils/generateClassCode.ts`. Characters O, 0, I, 1 are excluded to prevent visual confusion.

**AI tutor settings per assignment**: each assignment stores `voice` (formal/casual), `tone`, `evidence_required` (bool), and `content_source` (what the tutor can reference).

**Rubrics**: each assignment can have multiple rubric criteria (`assignment_rubrics` table). These drive the AI tutor's grading.
